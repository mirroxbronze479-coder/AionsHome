# 同步原作者更新教程

## 概念说明

你的仓库：`mirroxbronze479-coder/AionsHome`（你自己的，有你改的东西）  
原作者仓库：`death34018-hue/AionsHome`（上游，他的更新从这里来）

每次原作者更新了，你需要把他的更新合并到你的仓库里，同时保留你自己改的东西。

---

## 每次更新的流程

### 第一步：打开 PowerShell，进入项目目录

```
cd D:\AppData\AionsHome
```

### 第二步：拉取原作者的更新并合并

把下面这三行**一次性**复制粘贴进去运行：

```
git fetch upstream
git merge upstream/main
```

> `git remote add upstream ...` 只需要第一次运行，以后不用再加了。

---

### 第三步：看结果

**情况一：没有冲突**

PowerShell 显示类似这样的内容，说明合并成功，直接跳到第四步：

```
Merge made by the 'ort' strategy.
```

**情况二：有冲突**

PowerShell 显示类似这样的内容：

```
CONFLICT (content): Merge conflict in aion-chat/routes/chat.py
Automatic merge failed; fix conflicts and then commit the result.
```

有冲突的文件名会列出来，记下来，然后继续看下面。

---

### 第四步（有冲突时）：把冲突文件发给 Claude 处理

1. 去 `D:\AppData\AionsHome\aion-chat\` 找到冲突的文件
2. 直接把文件拖进 Claude 对话框发过来
3. 告诉 Claude："帮我解决冲突，我自己改的东西不要删"
4. Claude 改好后发回给你，下载文件，覆盖掉原来的文件

> 冲突文件里你会看到 `<<<<<<< HEAD`、`=======`、`>>>>>>> upstream/main` 这些标记，不用管，Claude 会处理。

---

### 第五步：提交并推送

冲突解决完之后，运行：

```
git add .
git commit -m "合并上游更新"
git push
```

推送成功后显示类似：

```
To https://github.com/mirroxbronze479-coder/AionsHome.git
   xxxxxxx..xxxxxxx  main -> main
```

就完成了。

---

## 注意事项

- **不要**在 GitHub 网页上点"弃掉 N 次提交"，那会删掉你自己的改动。
- **不要**在 GitHub 网页上点"开放拉取请求"，那是把你的东西推给原作者，方向反了。
- 每次合并前不需要做任何备份，Git 本身就是备份。
- 如果推送失败提示权限问题，检查远程仓库地址是不是你自己的（不是原作者的）：`git remote -v`
